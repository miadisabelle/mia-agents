## Common Collaboration Patterns

### Pattern 1: Starting a New Ceremony Spiral Project

**Step 1 - Cultural Foundation (Indigenous Knowledge Steward)**
- Validate community partnership and consent
- Establish cultural protocols for the project
- Define relational accountability framework
- Assess community benefit expectations

**Step 2 - Vision and Tension (Reciprocity Flow Analyst)**
- Facilitate community vision articulation
- Map current reality honestly
- Identify generative structural tension
- Design advancing spiral pattern

**Step 3 - Architecture Design (Ceremony Spiral Architect)**
- Create relationship-centered data models
- Design API with cultural protocol enforcement
- Plan GitHub integration approach
- Specify technical infrastructure

**Step 4 - Implementation Planning (GitHub Ceremony Integrator + MCP Coordinator)**
- Configure GitHub webhooks and integration
- Set up MCP server for ceremonies
- Implement ceremonial commit protocols
- Create collaborative digital spaces

**Step 5 - Documentation (Academic Research Advisor)**
- Develop theoretical framework document
- Create implementation guides
- Write community-facing documentation
- Prepare academic publication materials

### Pattern 2: Validating an Existing Implementation

**Validation Sequence:**
1. **Indigenous Knowledge Steward**: Cultural protocol compliance check
2. **Reciprocity Flow Analyst**: Relationship health assessment
3. **Ceremony Spiral Architect**: Technical architecture review
4. **GitHub Ceremony Integrator**: GitHub integration validation
5. **MCP Ceremony Coordinator**: Collaborative space functionality check
6. **Academic Research Advisor**: Theoretical alignment verification

### Pattern 3: Addressing a Challenge or Issue

**Diagnostic Approach:**

**If the issue is cultural/ethical:**
- Start with **Indigenous Knowledge Steward**
- Consult **Academic Research Advisor** for theoretical grounding
- Work with **Ceremony Spiral Architect** for structural changes

**If the issue is relational:**
- Start with **Reciprocity Flow Analyst**
- Consult **Indigenous Knowledge Steward** for cultural context
- Work with **MCP Ceremony Coordinator** for collaborative resolution

**If the issue is technical:**
- Start with **Ceremony Spiral Architect**
- Consult **GitHub Ceremony Integrator** or **MCP Ceremony Coordinator** as needed
- Validate with **Indigenous Knowledge Steward** for protocol compliance

### Pattern 4: Academic Article Development

## Common Questions

### "Which agent should I consult first?"

**For new projects**: Start with Indigenous Knowledge Steward to establish cultural foundation

**For technical questions**: Start with Ceremony Spiral Architect for overall design

**For relationship issues**: Start with Reciprocity Flow Analyst for pattern assessment

**For implementation**: Start with GitHub Ceremony Integrator or MCP Ceremony Coordinator depending on context

**For academic work**: Start with Academic Research Advisor for framework guidance

### "How do I engage multiple agents simultaneously?"

Describe your situation and explicitly request input from multiple agents. Example:

> "I'm designing a ceremony tracking system. I need:
> - @Ceremony-Spiral-Architect: Technical architecture guidance
> - @Indigenous-Knowledge-Steward: Cultural protocol validation
> - @Reciprocity-Flow-Analyst: Relationship modeling approach
> - @MCP-Ceremony-Coordinator: Real-time collaboration requirements"

### "What if agents disagree?"

Disagreement often reveals important tensions:

1. Clarify the source of disagreement
2. Consult Indigenous Knowledge Steward if cultural values are involved
3. Use Reciprocity Flow Analyst to assess whether tension is productive
4. Work with Ceremony Spiral Architect to design structural solution
5. Document the resolution process for future learning
